Главная причина поломок

Все 9 сломанных игр имеют одну и ту же корневую причину: неинициализированные переменные-массивы.

AI-генераторы объявляют переменные как let enemies, bullets, particles; и инициализируют их только в функции init(). Но цикл отрисовки requestAnimationFrame(draw) запускается сразу при загрузке страницы — до того, как пользователь нажмёт "Старт" и вызовет init(). В результате for (let e of enemies) падает с ошибкой TypeError: enemies is not iterable.

Рекомендация для промпта генерации: добавить в system prompt правило: "Always initialize array variables with = [] at declaration, not just inside init()."

Как задеплоить исправления

Исправленные HTML-файлы находятся в папке fixes/. Скрипт fix_broken_games.py обновит базу данных.

cd /path/to/GameDev/backend
python3 ../fix_broken_games.py

Скрипт автоматически создаст бэкап БД перед изменениями.

Рекомендация: предотвращение багов в будущем

Добавьте в system prompt генерации игр (backend/app/services/game_generator.py) строку:

CRITICAL: Always initialize ALL array and object variables at declaration time with default values (e.g. let enemies = [], let player = {x:0, y:0}).
Never declare variables with just 'let x, y, z;' if they will be used in for...of loops or have properties accessed before init() is called.
The draw/update loop starts via requestAnimationFrame immediately — before init() or startGame() is called by the user.
